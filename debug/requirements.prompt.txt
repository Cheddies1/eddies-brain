## Default Behaviour in This Folder

When I reference a transcript:

1. Do not summarise lazily.
2. Prioritise:
   - Decisions made.
   - Action items (with owner if identifiable).
   - Risks raised.
   - Open questions.
   - Misalignment or tension.
3. Highlight contradictions or unclear commitments.
4. Separate fact from interpretation.

If speaker attribution exists, preserve it where useful.  
If unclear, state that clearly.

If something is uncertain:
- Do not stop the summary.
- State the uncertainty explicitly in the output.
- Prefer omission over invention.
- Use phrases like:
  - "Owner unclear from transcript"
  - "Attendance unclear from transcript"
  - "Mentioned, not confirmed present"

Be concise but complete.

## Markdown Structure Standard

Use this structure:

# Meeting: [Title]

Date:  
Participants:
- List only people who clearly spoke in the transcript or are explicitly stated to be attending this meeting.
- Do not infer attendance from mentions, past meetings, or organisational context.
- If a person is discussed but not clearly present, put them under a separate line:
  Referenced:
  - Name – why they were mentioned
- Use organisational context only to correct obvious transcription errors in names, not to invent attendance or roles.
- If uncertain, omit the person from Participants and note the ambiguity under Open Questions only if it matters.

Audio: [[<same filename>.wav]]
Transcript: [[<same filename>.txt]]

Impact Level: Low / Medium / High
Commercial Exposure: Low / Medium / High

Confidence Level: Low / Medium / High

Confidence Level must reflect transcript clarity only, not model confidence.
If speaker attribution, attendance, or decisions are ambiguous, reduce Confidence Level.
If no major ambiguity exists, default to High.

## Meeting Summary Discipline

For transcript-to-note generation:

- Summarise what was actually said, not what would be sensible to say.
- Preserve the distinction between:
  - agreed decisions
  - proposed approaches
  - open questions
  - background context
- If the group says "probably", "I think", "it depends", or "we need to clarify", do not write it up as settled.
- If only one person suggests something and nobody aligns, treat it as discussion, not decision.
- If a person is only mentioned historically or externally, do not list them as attending.
